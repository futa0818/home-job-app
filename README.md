# 家庭内アルバイト管理アプリ（仮称）仕様書 Ver. 1.5

## 1. アプリの目的

* 家庭内のアルバイト（お手伝い）の申請・承認をデジタル化する。
* お手伝いの実績とお金を透明化し、月末の振り返りを簡単にする。
* アルバイトの内容や金額について、親子で提案・承認できる仕組みを作る。

## 2. 登場人物（アクター）

* 子（あなた）: アルバイトの実行と報告、新しいアルバイトや金額の提案。
* 親: 報告の確認とお金の管理、提案されたアルバイト内容・金額の審査。

## 3. 主要な機能一覧

* アルバイト申請・提案機能（子用）
* 実行の承認・メニュー審査機能（親用）
* 月末レポート機能（共通）

## 4. ターゲット端末とUIデザイン方針

* 対応端末: iOS（Expo Goを通じたiPhoneでの動作）
* デザインコンセプト: 洗練されたダッシュボード風UI。整理されたカッコいい画面を目指す。

## 5. 技術スタック・開発環境

* コードエディタ: VS Code
* フロントエンド: React Native + Expo (TypeScript / JavaScript)
* 実機検証: Expo Go (iPhone)
* バックエンド: Firebase (Cloud Firestore)
* バージョン管理: GitHub

## 6. 画面構成（スクリーン設計）

| 画面名     | ユーザー | 主な役割とレイアウト                                                                                         |
| ------- | ---- | -------------------------------------------------------------------------------------------------- |
| ログイン画面  | 共通   | アプリ起動時のスタート画面。「子として入る」「親として入る」のボタンを配置。                                                             |
| 子用メイン画面 | 子    | 最上部に「今月の合計金額」を大きく表示。その下の「詳細」ボタンを押すとスライドして開き、「今月実施したアルバイトの内容と履歴」が一覧で確認できる。 画面下部には申請・提案ボタンをタイル状に並べる。 |
| 親用メイン画面 | 親    | 子から届いた「承認待ち」のリストが並ぶ。スワイプやワンタップで「承認」「却下」をスピーディーに捌けるUI。                                              |
| レポート画面  | 共通   | 今月の合計金額や、承認済みの履歴リストが確認できる画面。タブ等でメイン画面と切り替える。                                                       |
