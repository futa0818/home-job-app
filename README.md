# 家庭内アルバイト管理アプリ 仕様書・設計書

## 1.アプリの目的
    
- 家庭内のアルバイト（お手伝い）の申請・承認をデジタル化し、親子間のやり取りをスムーズにする。
- お手伝いの実績と金額を透明化し、月末の振り返り（お小遣い計算）を簡単にする。
- アルバイトの内容や金額について、親子で提案・交渉・承認できる仕組みを作る。
## 2.登場人物（アクター）
    
- 子：アルバイトを実行し申請する。新しいアルバイト項目や金額アップの提案を行う。
- 親：子の申請を確認し、承認/却下してお金を管理する。提案されたメニューの審査を行う。
## 3.主要な機能一覧
    
- アルバイト申請・提案機能（子用）
- 実行の承認・メニュー審査機能（親用）
- 月末レポート機能（共通）
## 4.デザイン方針

- UI/UXデザイン: 「洗練されたダッシュボード風UI」。ごちゃごちゃさせず、直感的に操作できるカッコいい画面。アコーディオン（スライド開閉）などを活用し、画面をスッキリ保つ。
## 5.画面構成（スクリーン設計）

| 画面名     | 利用者 | 主な機能とレイアウト                                                                                                                  |
| ------- | --- | --------------------------------------------------------------------------------------------------------------------------- |
| ログイン画面  | 共通  | - アプリ起動時のスタート画面。<br>- 「子として入る」「親として入る」を選択。                                                                                  |
| 子用メイン画面 | 子   | - 最上部に「今月の合計金額」を大きく表示。<br>- 合計金額の下の「詳細」ボタンを押すとスライドで「今月の履歴」が表示される。・画面下部にアルバイトの申請ボタン（タイル状）を配置。<br>- 「新しいアルバイト/金額を提案する」ボタンを配置。 |
| 親用メイン画面 | 親   | - 子から届いた「承認待ちの申請」「提案」をリスト表示。<br>- スワイプやワンタップで「承認」「却下」をスピーディーに処理できるUI。                                                       |
| レポート画面  | 共通  | - 今月の合計金額や、承認済みの履歴リストが確認できる。過去の月の履歴も参照可能。                                                                                   |
| 設定画面    | 共通  | - プッシュ通知のオン/オフ切り替えなどを行う。                                                                                                    |
