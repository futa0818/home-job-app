# 家庭内アルバイト管理アプリ 仕様書・設計書

## 1. アプリの目的
* 家庭内のアルバイト（お手伝い）の申請・承認をデジタル化し、親子間のやり取りをスムーズにする。
* お手伝いの実績と金額を透明化し、月末の振り返り（お小遣い計算）を簡単にする。
* アルバイトの内容や金額について、親子で提案・交渉・承認できる仕組みを作る。

## 2. 登場人物（アクター）
* 子：アルバイトを実行し申請する。新しいアルバイト項目や金額アップの提案を行う。
* 親：子の申請を確認し、承認/却下してお金を管理する。提案されたメニューの審査を行う。

## 3. 主要な機能一覧
* アルバイト申請・提案機能（子用）
* 実行の承認・メニュー審査機能（親用）
* 月末レポート機能（共通）

## 4. ターゲット端末とデザイン方針
* 対応端末: iOS（親子のiPhoneで利用。Expo Goアプリ上で動作）
* UI/UXデザイン: 「洗練されたダッシュボード風UI」。ごちゃごちゃさせず、直感的に操作できるカッコいい画面。アコーディオン（スライド開閉）などを活用し、画面をスッキリ保つ。

## 5. 技術スタック（開発環境）
* コードエディタ: VS Code
* フロントエンド: React Native + Expo (TypeScript or JavaScript)
* 実機テスト環境: Expo Go
* バックエンド・DB: Firebase (Firebase Authentication and Cloud Firestore)
* バージョン管理: GitHub

## 7. データベース設計（Firebase Cloud Firestore）

Firebaseでは、データを「コレクション（フォルダのようなもの）」と「ドキュメント（ファイルのようなもの）」に分けて保存します。以下がこのアプリに必要な4つの主なコレクション構造です。

### ① users コレクション（ユーザー情報と設定）

誰がアプリを使っているか、通知設定はどうなっているかを保存します。

* role: "parent" または "child"
* notificationsEnabled: true / false （通知のオンオフ）
* pushToken: 通知を送るための端末専用の鍵（文字列）

### ② jobs コレクション（アルバイトのメニュー・マスターデータ）

現在有効なアルバイトの種類と金額のリストです。（初期データとして以下を登録）

* name: "洗濯物干し", "洗濯物畳み", "風呂洗い", "靴並べ・チェーンしめ"
* reward: 50, 50, 50, 30 （金額）
* isActive: true （現在使えるメニューかどうか）

### ③ requests コレクション（日々のアルバイト申請データ）

子が「今日これをやった！」と送信した履歴データです。

* jobName: "洗濯物干し" （やったこと）
* reward: 50 （その時の金額）
* status: "pending" (承認待ち) / "approved" (承認済み) / "rejected" (却下)
* createdAt: 2026-06-27 18:00:00 （申請した日時）

### ④ proposals コレクション（新しいアルバイトや金額の提案データ）

子が親に「これをお小遣いメニューに追加して！」と提案したデータです。

* type: "new_job" (新規追加) または "update_reward" (金額変更)
* jobName: "草むしり" （提案する内容）
* proposedReward: 100 （提案する金額）
* status: "pending" (審査待ち) / "approved" (採用！) / "rejected" (不採用)
* createdAt: 提案した日時
